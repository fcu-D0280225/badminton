# 設計系統 — 羽球館方管理 PWA

> Generated by `/design-consultation` · 2026-04-01

---

## 定位

這是台灣小型羽球場館館主的**日常操作工具**，不是給顧客看的行銷頁面。  
用戶一天查看 10 次以上，通常是站在球場邊拿手機看。  
設計原則：**緊湊、行動優先、關鍵數字一眼就看到。**

---

## 美學方向

**Compact Operational（緊湊實用主義）**

- 工具優先，資訊密度高，無裝飾
- 比 SaaS 感更接近每天用的 LINE 介面
- 不用大留白來「看起來高級」，用好的資訊階層讓每個像素都有意義

**Noto Sans TC** — 覆蓋繁體中文 + 數字，保留既有設定。

| 用途 | 大小 | 字重 | 備註 |
|------|------|------|------|
| Hero 金額（未收款） | 28px | 700 | `tabular-nums`，紅色 |
| 頁面 h2 | 20px | 700 | — |
| Section h3 | 16px | 500 | — |
| 卡片金額 | 15px | 700 | `tabular-nums` |
| 隊伍名稱 | 14px | 600 | — |

## 動畫 / 動效

**最小功能性** — 只有助於理解的轉場，不加裝飾動畫。

- Modal 進場：`transform translateY(-30px) → 0` + `opacity 0 → 1`，200ms ease-out（已有）
- Tab 切換：無動畫（即時）
- 按鈕 hover：`translateY(-1px)` + box-shadow（桌機只）
- 不加 loading spinner、skeleton、進場動畫

## 核心設計決策（Risks）

### R1 — 未收金額英雄化

**做什麼：** 收費 Tab 最頂部放一張「英雄卡」，顯示本月未收總額（28px bold 紅色），以及已收進度。

**為什麼：** 場館老闆打開 app 只有一個問題：「今天還有多少錢沒收到？」讓這個數字成為第一眼看到的東西，其餘清單是「確認工具」不是「主角」。

**實作位置：** `app.js` → `renderBillingSummaryBanner()` 重寫為 `renderBillingHero()`。從 analytics API 取得當月真實數字。

```html
<div class="billing-hero">
  <div class="billing-hero-amount">NT$12,400</div>
  <div class="billing-hero-subtitle">3 筆未付款 · 本月共 NT$28,000</div>
  <div class="billing-hero-paid-row">
    <span>已收款</span>
    <span class="billing-hero-paid-amount">NT$15,600 ✓</span>
  </div>
</div>
```

### R2 — 卡片左邊框狀態色條

**做什麼：** 每張收費卡片左側加一條 4px 顏色條（紅/綠/黃），替代或輔助文字徽章。

**為什麼：** 大腦識別顏色比閱讀文字快。快速滾動清單時，眼角就能掃到紅色（未付）不需要對焦。

**實作位置：** `styles.css` → `.billing-card` + modifier classes。

```css
.billing-card { display: flex; overflow: hidden; border-radius: var(--radius-card); }
.billing-card .card-status-bar { width: 4px; flex-shrink: 0; }
.billing-card.status-unpaid  .card-status-bar { background: var(--status-unpaid-border); }
.billing-card.status-paid    .card-status-bar { background: var(--status-paid-border); }
.billing-card.status-transfer .card-status-bar { background: var(--status-transfer-border); }
```

### R3 — 清單字體縮小至 14px

**做什麼：** 一般內文從 16px 改為 14px，卡片間距從 15px 改為 8px。

**為什麼：** 場館老闆用現代手機（高解析度螢幕）。14px 完全可讀，但可以讓一個畫面多顯示 2-3 筆記錄，減少捲動次數。

**注意：** 只縮小清單 / meta 文字。表單 label 保留 14px，按鈕保留 16px，金額保留 15px 以上。
